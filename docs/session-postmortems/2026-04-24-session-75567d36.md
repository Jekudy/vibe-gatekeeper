## Process Gaps Identified by Meta-Critics

### Reality Checker (Meta-critic #1)

1. **No evidence gate** — orchestrator accepts agent DONE at face value. Fix: every DONE must include raw command output.
2. **Scope creep** — 23 tasks for a 3-batch sprint (realistic: 15-17).
3. **Overloaded prompts** (500+ lines) → agents lose focus, produce "comprehensive" reports instead of targeted work.
4. **Dependency graph inflated** — E3 blocked by 7 tasks, real hard deps: 2-3.
5. **No pre-dispatch user sync** — for secrets/naming, ask format BEFORE dispatching agent.
6. **P0/P1 mislabeling** — Z4 (production broken 4 days) initially classified as "side finding".
7. **No agent files_modified registry** — impossible to trace which agent modified which file.
8. **Security agent not evidence-gated** — Z1 said "checked, clean" without grep output. Orchestrator didn't request retry.

## Proposed Process Changes

### P1: Evidence Gate (prevents errors #1-6, #8, #10-11, #17-18)
Every agent DONE claim must include raw command output. Orchestrator verifies evidence before marking complete. "Trust but verify" → "verify then trust".

### P2: Post-Deploy Smoke Test Checklist (prevents #7-8)
After any Coolify deploy: check Mounts, check container file exists, check logs for errors (not just "running").

### P3: Pre-Dispatch User Sync (prevents #12-13)
For any task involving secrets, naming, or user-facing values: ask format/preference BEFORE dispatching agent.

### P4: Agent Prompt Size Limit (prevents #14-16)
Cap agent prompts at 200 lines. Split large tasks into smaller, focused sub-tasks.

### P5: Working Tree Hygiene Gate (prevents #22-23)
Before any commit: `git status` + verify no foreign files. After tests: verify counter/state returned to baseline.

### P6: Security Evidence Standard (prevents #9-11)
Security review agent must return raw `git grep` / `grep -r` output. "Checked" without evidence = NOT checked.
